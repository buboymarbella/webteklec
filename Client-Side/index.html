HTML

HTML (HyperText Markup Language) is the most basic building block of the Web. It describes and defines the content of a webpage. Other technologies besides HTML are generally used to describe a webpage's appearance/presentation (CSS) or functionality (JavaScript). HTML is a markup language (not a programming language) that is used to create structured documents (i.e. web pages) in the World Wide Web. It was introduced by the rules of Standard Generalized Markup Language (SGML) in document type definition (DTD), by Tim-Berners-Lee in the year 1991. Creates structured documents by denoting semantics for text like headings, lists, links, quotes etc.

HTML Versions

Version Description
HTML 1.0 HTML 1.0 was the first release of HTML to the world. Not many people were involved in website creation at the time, and the language was very limiting. There really wasn’t much you could do with it bar getting some simple text onto the web. But then, just that got the beardos a-foamin’ back in the day.
HTML 2.0 HTML 2.0 included everything from the original 1.0 specifications but added a few new features to the mix. HTML 2.0 was the standard for website design until January 1997 and defined many core HTML features for the first time.
HTML 3.0 It included many new and improved abilities for HTML, and promised far more powerful opportunities for webmasters to design their pages. Sadly, the browsers were awfully slow in implementing any of the new improvements, only adding in a few and leaving out the rest. Partly, this failure can be attributed to the size of the overhaul and so the HTML 3.0 spec was abandoned.
HTML 3.2 This was a toned-down change to the existing standards, leaving many of the big steps forward for later versions. Most of the extensions tags that had been introduced by Netscape (and to a lesser-extent, Microsoft) did not make it into these new standards. It soon caught on and became the official standard in January ’97, and today practically all browsers support it fully.
HTML 4.0 Extends HTML with mechanisms for style sheets, scripting, frames, embedding objects, improved support for right to left and mixed direction text, richer tables, and enhancements to forms, offering improved accessibility for people with disabilities.
HTML 4.01 This version is strict, transitional and uses frameset.
HTML 5 Is the fifth and current version of the HTML standard.
HTML 5.1 Is the first minor revision of HTML 5 and is currently under Proposed Recommendation since September 2016.

HTML VS. XHTML

HTML and XHTML are quite related to each other but here are their possible differences.


HTML XHTML
Start or end tags are not required for every element. All elements must have a start tag.
Tags and attributes are case-insensitive. Elements and attributes are case sensitive, it is mostly in lowercase.
Attribues do not need to be quoted. Attribute values must be enclosed in quotes.
The document must include an HTML5 DOCTYPE. XHTML documents must be well-formed.
All the content can be put under the body element. All elements are nested properly.

CSS

CSS History and Origin

CSS or cascading style sheetsspan> is used for describing the presentation of an html document. Though CSS is not limited to only HTML documents it may also be applied to SVG, XML and XHTML docs.

CSS was proposed by Hakon Wium Lie at around that time several proposals for style sheet languages was proposed. This led to the World Wide Web Consortium's first CSS Recommendation. As his proposal was influential Bert Bos became co-author of CSS1


CSS Variations

  • CSS 1 - includes font properties, line spacing and text attributes, text colors, backgrounds and positioning options such as margin and paddings.
  • CSS 2 - added capabilities for positioning of elements such as relative, absolutes and Z - index. Media type concepts and font properties.
  • CSS 2.1 - This first revision of the CSS 2 moves forward by fixing errors and removing not fully supported features from the previous version.
  • CSS 3.0 - Preserves backwards compatibility. Now divided into modules.

CSS Selectors

In CSS in order to refer to an element that we want to style we need to indicate a selector. This can be done in different ways with different selectors.

Selector Syntax Description
type p Selects a specific element to style
class .proj Selects an element with a matching class attribute
id #proj Selects an element with a matching id attribute
* * Selects all elements

CSS Combinators

To further specify the element you want to style combinators offer a lot of variations in selecting an element.

Combinators Syntax Description
Descendant Selector A B This combinator selects children of the first specified element even if not direct descendants
Child Selector A > B The combinator selects children that are direct descendants of the first specified element
General Sibling Selector A ~ B This combinator selects the latter element that follows after the former element not necessarily immediately
Adjacent Sibling Selectors A + B This combinator selects the latter element that follows immediately after the former element

CSS Psuedo Elements

A Psuedo element allows the user to style specific part of a selected element.Some of them are:

  • ::after - This will be placed after the element.
  • ::before - This will be placed before the element.
  • ::first-letter - Only applies to the first letter of the element.
  • ::first-line - Only applies to the first line of the element.
  • ::selection - Applies to a portion of a document highlighted by the user.
